# Phase 7：真实浏览器验收与体验收尾

## 1. 目标与范围

Phase 7 使用本地 Vite 服务和 Codex 内置真实浏览器，对 v2.4 辅助运算进行端到端验收。重点验证自动化单元测试不容易覆盖的真实渲染、路由切换、刷新恢复、响应式布局、动画节奏、键盘操作和浏览器控制台状态。

本阶段不重复验证 Phase 6 已覆盖的纯函数和组件内部算法。发现缺陷时先记录复现步骤、预期结果和实际结果，再修改代码；修改后必须重新执行受影响场景以及全量自动化测试。

## 2. 验收环境

### 2.1 启动前检查

在项目根目录执行：

```bash
git status -sb
npm test -- --run
npm run build
```

预期结果：

- 工作区没有来源不明的改动。
- 全量测试全部通过。
- Vite 生产构建成功，无编译错误。

### 2.2 启动本地服务

```bash
npm start -- --host 127.0.0.1
```

浏览器访问：

```text
http://127.0.0.1:5173/that-math-things/
```

预期结果：

- Vite 服务成功监听本机地址。
- 首页正常显示，不出现白屏、404 或资源加载失败。
- 验收只访问本机 `127.0.0.1`，不需要部署或暴露公网端口。

### 2.3 浏览器操作通用规则

每次浏览器交互都按以下顺序执行：

1. 获取当前页面结构或截图，确认页面处于预期路由和状态。
2. 使用按钮名称、输入框占位符、稳定链接或稳定属性定位元素。
3. 定位结果不唯一时先缩小到对应卡片或区域，不按猜测位置点击。
4. 执行一次点击、输入、按键、刷新或尺寸调整。
5. 立即读取变化后的局部状态；只有下一步需要重新定位时才获取完整页面结构。
6. 每个主要流程结束后读取浏览器 `warning` 和 `error` 级别控制台日志。

禁止用固定坐标代替可识别的页面元素，除非页面结构无法提供有效定位。不要通过读取或直接修改浏览器 localStorage 来伪造验收结果，设置和记录必须通过真实界面产生。

## 3. 测试数据与执行约定

- 每轮使用 10 题，降低完整流程耗时；20 题和 50 题只验证设置保存、题数展示和首屏布局。
- 题目是随机生成的。需要特定题型时，提高进位/退位难度，并通过正常答题继续寻找目标题，不直接篡改题目数据。
- 至少覆盖一题进位加法、一题退位减法和一题无需辅助的简单题。
- 至少故意答错一题，以产生结算错误项和订正入口；其余题目按正确答案提交。
- 破十法和平十法分别开始一轮训练，至少各完整查看一次退位动画。
- 验收记录中写明实际遇到的题目，例如 `19+24`、`32-24`，不得只写“进位题”或“退位题”。

## 4. 验收场景

### 4.1 首页与设置页基础流程

浏览器指令：

1. 打开本地地址。
2. 从首页进入“计算训练”。
3. 确认设置页显示运算范围、加法比例、进位/退位难度、退位个位算法、辅助运算、题目数量、开始训练和统计数据。
4. 依次选择 10 题、20 题和 50 题，观察选中状态。
5. 将辅助运算关闭，再观察破十法和平十法选项。
6. 重新开启辅助，分别选择破十法和平十法。

预期标准：

- 页面没有横向溢出，标题、说明和控件不重叠。
- 题数选项每次只有一个处于选中状态。
- 关闭辅助后，退位个位算法不可操作，但当前值不发生异常跳变。
- 开启辅助后，破十法和平十法可以正常切换，说明文字完整可读。
- 刷新设置页后，最近一次设置仍能恢复。

### 4.2 关闭辅助的原有做题流程

浏览器指令：

1. 在设置页关闭“辅助运算”。
2. 选择 10 题并点击“开始训练”。
3. 检查题目、输入框、下一题按钮、进度和计时器。
4. 连续完成至少三题，其中一题使用按钮提交，一题使用 Enter 提交。
5. 观察每次切题后的输入框焦点和计时器。

预期标准：

- 所有题目均不显示“需要提示”入口和辅助演示区域。
- 输入数字后“下一题”可用，空输入不能提交。
- Enter 与点击按钮行为一致，每次只前进一题，不重复提交。
- 切题后输入框清空并重新获得焦点。
- 计时器连续递增，不倒退、不跳回零、不因输入而异常加速。
- 题目切换时页面不闪烁，不出现旧题辅助状态。

### 4.3 刷新恢复

浏览器指令：

1. 在设置页选择一组容易识别的设置，例如 20 以内、10 题、关闭辅助。
2. 点击“开始训练”，确认已进入第 1 题。
3. 直接刷新浏览器页面。
4. 检查刷新后的路由、题目和辅助状态。
5. 返回设置页，确认设置值仍为刷新前选择的值。

预期标准：

- 刷新后仍停留在练习页，不白屏，也不因缺少路由 state 崩溃。
- 页面可从持久化设置重新建立一场训练。
- 关闭辅助的状态保持一致，不出现辅助入口。
- 当前实现不承诺恢复刷新前已经答到的题号；若重新生成题目，应作为已知行为记录而不是缺陷。

### 4.4 第一层提醒

浏览器指令：

1. 开启辅助，选择最高进位/退位难度并开始 10 题训练。
2. 遇到符合条件的题目时，先观察未点击状态。
3. 点击“需要提示”。
4. 只阅读第一层内容，不立即打开完整方法。
5. 收起后再次展开。

预期标准：

- 简单且不涉及真实进位或退位的题目不显示辅助入口。
- 真实进位或退位题显示低干扰的“需要提示”，但未点击前不泄露步骤。
- 第一层只提示进位或退位关键点，不直接显示最终答案。
- 收起和再次展开操作稳定，题目、输入内容和计时不受影响。
- 第一层使用记录应在最终摘要中归入“只看提醒”。

### 4.5 进位加法完整演示

浏览器指令：

1. 在进位加法题点击“需要提示”，再点击“看看计算方法”。
2. 在第一步确认两个加数按上下两行、十位和个位分别显示。
3. 点击“下一步”，观察个位先合并为 10 个一和剩余个位。
4. 再点击“下一步”，观察 10 个一换成 1 个十并移动到十位。
5. 再点击“下一步”，观察十位中的两个原始数量与进来的 1 合并。
6. 使用“上一步”逐步返回，再使用“下一步”前进。
7. 使用“重播”重新从第一步开始，最后使用“返回做题”或完成演示。

预期标准：

- 第一步不提前把两个加数的十位或个位混成一组。
- 个位融合、满十进一、十位融合是可辨认的连续过程。
- 十位原始数量在个位运算期间保持不动；进来的 1 有明确来源。
- 进位后剩余个位保持显示，最终数位与计算结果一致。
- 上一步能恢复前一状态，反复前后切换不会丢失或重复小棒。
- 演示完成后仍由用户填写答案，动画不自动代填。

### 4.6 退位减法——破十法

浏览器指令：

1. 设置“退位个位算法”为“破十法”，开始训练。
2. 在退位题打开完整方法演示。
3. 观察被减数和减数的十位、个位保持上下对应。
4. 前进到借位步骤，确认 1 个十换成 10 个一。
5. 前进到个位步骤，确认按“先算 10−减数个位，再加回原个位”的顺序展示。
6. 前进到十位步骤，确认个位结果继续保留。
7. 使用上一步、下一步和重播复核状态可逆性。

预期标准：

- 借位来源清楚：十位减少 1，个位新增完整的一组 10 个一。
- 10 个一按一排 10 个显示，不以 9 个一换行造成误读。
- 被划去的小棒和保留的小棒视觉上能区分。
- 个位最终结果明确显示，不只展示中间的 `10−b`。
- 进入十位计算后，已经得到的个位结果保持不变。
- 十位相减和最终答案与题目一致。

### 4.7 退位减法——平十法

浏览器指令：

1. 返回设置页，将“退位个位算法”切换为“平十法”。
2. 开始新一轮训练，在退位题打开完整方法演示。
3. 观察减数个位拆分为“先减到整十”和“再减剩余部分”。
4. 逐步前进并使用上一步返回一次。
5. 完成演示后自行输入答案。

预期标准：

- 动画实际使用平十法，不只修改说明文字。
- 例如 `12−4` 应表达为 `12−2=10`，再算 `10−2=8`。
- 十位和个位的原始上下关系在方法开始时保留。
- 个位最终结果在后续十位步骤中持续显示。
- 破十法与平十法只改变退位个位的大步骤，加法动画不受设置影响。

### 4.8 临界题与边界题

遇到下列题型时按实际题目记录结果：

- `18+2` 一类个位和正好为 10 的加法。
- `10−3` 一类被减数正好为 10 的减法。
- 整十数参与的退位题。
- 接近当前运算范围上限的题目。

预期标准：

- `18+2` 属于真实进位，不能因剩余个位为 0 而排除辅助。
- `10−3` 可以提供减法辅助；省略的前置或后置步骤必须明确说明，不能生成不存在的运算。
- 不出现负数小棒数量、重复借位、空白步骤或与题目不符的公式。
- 运算结果不能超过所选范围约束。

### 4.9 自动播放速度与手动控制

浏览器指令：

1. 分别选择“快 5 秒”“中 10 秒”“慢 20 秒”。
2. 每档从第一步开始播放，记录自动进入下一步的大致时间。
3. 自动播放期间点击“上一步”和“下一步”。
4. 点击“跳过演示”，随后重新打开并重播。

预期标准：

- 三档选中状态清楚，同一时间只有一档生效。
- 自动推进间隔分别约为 5、10、20 秒，允许少量浏览器调度误差，但不能仍以旧的 1.8 秒推进。
- 手动切换后自动计时从当前步骤合理重新开始，不立即连跳两步。
- 第一步的“上一步”和最后一步的“下一步”禁用或具有明确结束行为。
- 跳过、重播和返回做题不会修改已输入的答案。

### 4.10 完整结算、历史与订正闭环

浏览器指令：

1. 完成一轮 10 题训练：至少一题不使用辅助、一题只看第一层、一题查看第二层，并故意答错至少一题。
2. 在最后一题分别验证 Enter 或“完成”按钮进入结算页。
3. 核对得分、正确数、错误数、用时和辅助使用摘要。
4. 进入统计数据或历史记录，打开刚完成的记录。
5. 从记录进入错题订正。
6. 在订正输入框输入正确答案并按 Enter，直至完成订正。

预期标准：

- 最后一题只保存一次，不产生重复历史记录。
- 得分、正确数、错误数与实际作答一致。
- 辅助摘要中的“独立完成”“只看提醒”“查看方法”与本轮实际操作一致；普通题不进入辅助摘要分母。
- 历史详情能正常读取新结构记录，不出现字段缺失或 `[object Object]`。
- 订正只包含本轮错题，Enter 可提交，正确后进入下一题或显示“全部订正完成”。
- 订正流程不新增或篡改原练习的辅助使用记录。

## 5. 响应式尺寸验收矩阵

每个尺寸都从页面顶部、设置页和一张展开的辅助演示卡片各检查一次。

| 视口宽度 | 浏览器操作 | 预期结果 |
|---|---|---|
| 767px | 将视口调整为 767px | 显示“目前网站只支持电脑和 Pad 访问”，训练内容不可操作 |
| 768px | 从 767px 调整为 768px | 拦截层消失，应用立即恢复，不需要手动刷新 |
| 1024px | 调整为 1024px | 设置、题目和演示卡片完整显示，无横向滚动和遮挡 |
| 1440px | 调整为 1440px | 内容保持合理最大宽度，不被无意义拉伸，操作区仍居中 |

额外检查：

- 在 768px 下展开最长的退位演示，按钮、公式和小棒不得越出卡片。
- 从宽屏缩到 767px，再恢复到 1024px，页面必须能在不刷新情况下恢复。
- 视口切换不能重置当前题号、答案输入或辅助演示步骤。

## 6. 减少动态效果与可访问性

### 6.1 减少动态效果

在浏览器或操作系统中开启“减少动态效果”后重新进入演示。

预期标准：

- 当前步骤直接呈现稳定终态，不依赖移动过程才能理解。
- 前后切换、跳过、重播和自动播放仍可操作。
- 不出现元素停在半路、透明度为零或内容永久缺失。

### 6.2 键盘操作

浏览器指令：

1. 使用 Tab 在设置控件、开始训练、答案输入框和辅助按钮之间移动。
2. 使用 Space 或 Enter 激活当前按钮。
3. 在答案输入框按 Enter 提交。

预期标准：

- 焦点顺序与视觉顺序基本一致，当前焦点可辨认。
- 不会被困在动画卡片内部。
- 输入框、下一题、上一步、下一步、跳过和重播均能通过键盘访问。
- 一次按键只触发一次动作。

## 7. 控制台与运行状态检查

在以下节点读取浏览器控制台：

1. 首页首次加载后。
2. 展开一次进位或退位完整演示后。
3. 刷新练习页后。
4. 进入结算页后。
5. 完成订正后。

通过标准：

- 没有未捕获异常、React 渲染异常、路由错误或资源 404。
- 没有新增的 React key、状态更新、卸载后更新或可访问性警告。
- Vite 开发环境自身的信息日志不计为缺陷。
- 对无法确认来源的 warning 必须记录完整文本、所在路由和触发步骤。

## 8. 缺陷记录格式

每个问题按以下格式记录：

```markdown
### P1/P2/P3：问题标题

- 环境：浏览器、视口、辅助设置、退位算法
- 题目：例如 32−24
- 前置条件：当前题号和已展开层级
- 操作步骤：按实际点击顺序逐条列出
- 预期结果：应看到什么
- 实际结果：实际看到什么
- 控制台：无 / warning 或 error 原文
- 截图：文件路径
- 复现率：必现 / 偶现，出现次数
```

优先级定义：

- `P1`：答案、算法步骤、存储数据错误，或流程无法继续。
- `P2`：动画状态、刷新、键盘、布局存在明显问题，但有替代路径。
- `P3`：文字、间距、颜色、节奏等不影响正确性的体验问题。

## 9. 修复后的回归顺序

发现并修复问题后依次执行：

1. 重跑直接相关的自动化测试文件。
2. 重跑该问题对应的真实浏览器场景。
3. 回归同类的加法或减法动画，防止共享组件受到影响。
4. 执行 `npm test -- --run`。
5. 执行 `npm run build`。
6. 再次检查浏览器控制台。
